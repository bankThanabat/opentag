# OpenTag CLI Quickstart Video Design

## Style Prompt

A concise technical tutorial for a local developer tool. The visual language should match OpenTag's existing README hero: dark canvas, warm gold signal lines, quiet terminal surfaces, and clear product flow. It should feel precise and practical, not like a launch trailer.

## Colors

- Background: `#0A0F14`
- Panel: `#111A20`
- Raised panel: `#172229`
- Text: `#F4EFE5`
- Muted text: `#A9B4AE`
- OpenTag gold: `#F5B83D`
- Success green: `#75C889`

## Typography

- Display and body: `JetBrains Mono` for terminal-native clarity, with system sans fallback for renderer safety.
- Commands and IDs: `JetBrains Mono`
- Use large, readable text. This video is a tutorial, so clarity beats decorative type.

## Motion

- Medium-energy tutorial pacing.
- Use blur crossfades between scenes.
- Animate command lines and choices in the order a user would experience them.
- Keep ambient motion subtle: signal lines, terminal cursor, and small status pulses.

## What NOT To Do

- Do not use neon purple or cyan gradients.
- Do not turn the video into a broad product manifesto.
- Do not show commands that are not currently supported by the published CLI.
- Do not expose fake secrets, real tokens, or long setup forms.
- Do not crowd the frame with architecture details.
