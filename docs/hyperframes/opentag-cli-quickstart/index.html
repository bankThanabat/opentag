OpenTag CLI

Run coding agents from your work threads.

One setup command connects Lark, Slack, or GitHub to a local coding agent.

local terminal
$ npm install -g @opentag/cli
$ opentag setup
Setup asks what to connect, then writes one local config.
Step 1

Run the setup wizard.

Pick the platform, the coding agent, and the local project OpenTag may work on.

Lark / Feishu Scan a Personal Agent QR code or reuse saved credentials.
Slack Use Socket Mode locally or Events API for public endpoints.
GitHub Use a repository webhook and token for comments and PRs.
Step 2

Start the local OpenTag stack.

The CLI starts the dispatcher, runner, and selected platform listener in one process.

✓dispatcher online
✓runner bound to this project
✓platform listener ready
$ opentag start
Press Ctrl-C to stop.
Step 3

Mention OpenTag where work already happens.

Slack
@OpenTag summarize this thread and propose the next step.
OpenTag replies in the same thread.
GitHub
@opentag investigate this failing test.
OpenTag can comment, push a run branch, and suggest a PR.
Lark / Feishu
@OpenTag check the latest local changes.
The local coding agent runs on your machine.
Safe actions

Approve important writes from the same thread.

OpenTag suggests
create_pull_request
Includes branch, changed files, title, and risk summary.
→
User replies
apply 1
OpenTag creates the PR only after the thread approval.
